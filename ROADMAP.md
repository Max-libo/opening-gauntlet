# Roadmap

## Фазы разработки

---

## Phase 0 — Foundation ✅ DONE
**Цель:** Рабочая шахматная доска с Stockfish в браузере

### Задачи
- [x] Инициализация проекта (Vite + TypeScript)
- [x] Интеграция chess.js — базовая логика ходов
- [x] Рендер шахматной доски (HTML/Canvas/SVG — DOM+CSS)
- [x] Подключение Stockfish WASM через WebWorker (stockfish-18-lite-single)
- [x] Проверка: игрок ходит белыми, Stockfish отвечает
- [x] Базовые стили, минималистичный UI (тёмная тема, eval bar, лог ходов)

**Результат:** Можно сыграть партию против Stockfish.
**Критерий готовности:** Партия проходит до мата без ошибок.

---

## Phase 1 — Opening System (Система дебютов)
**Цель:** Игра «знает» теорию и показывает это игроку

### Задачи
- [ ] Загрузка Opening Book (локальный JSON)
- [ ] Детектор: текущий ход — теоретический или нет
- [ ] Визуальный индикатор на доске (цвет рамки хода)
- [ ] Knowledge Bar — счётчик теоретических ходов
- [ ] ИИ ходит по книге (Opening Book), потом переключается на SF
- [ ] Индикатор «вышел из теории» с номером хода

**Результат:** Видно, насколько хорошо ты знаешь дебют.

---

## Phase 2 — Enemy System (Система противников)
**Цель:** Первые 3 противника с портретами и репликами

### Задачи
- [ ] Интерфейс Enemy (data structure + JSON)
- [ ] Экран встречи с противником (портрет, имя, реплика)
- [ ] Противник играет свою Opening Line (не произвольный SF)
- [ ] Реплики появляются в нужные моменты партии
- [ ] Создать 3 первых противника: Итальянец, Испанец, Сицилийка
- [ ] SVG-портреты для этих трёх
- [ ] Разные уровни SF для разных противников

**Результат:** Три уникальных «персонажа» с характером.

## Phase 3 — Roguelike Loop (Петля рогалика)
**Цель:** Полноценный ран от начала до Game Over

### Задачи
- [ ] GameState — централизованное состояние
- [ ] RunManager — генерация и прогрессия рана
- [ ] Система HP (❤❤❤)
- [ ] Экран между раундами (награды, следующий враг)
- [ ] Экран Game Over с итогами
- [ ] Главное меню
- [ ] Переход между этажами (5 врагов в ране)
- [ ] Сохранение прогресса в localStorage

**Результат:** Игру можно пройти от начала до конца.

**Цель:** Система карт и прогрессия между ранами

### Задачи
- [ ] Дизайн карты дебюта (HTML/CSS)

## Принципы разработки

1. **Playable First** — каждая фаза даёт рабочий билд
2. **No Premature Abstraction** — не усложнять то, что не нужно сейчас
3. **Content is Data** — противники, реплики, дебюты в JSON, не в коде
4. **Offline First** — игра работает без интернета (кроме опционального Lichess API)
